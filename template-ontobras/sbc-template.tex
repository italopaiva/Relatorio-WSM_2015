\begin{abstract}
  
  The brazilian federal highway police (Polícia Rodoviária Federal, in portuguese) have provided data about the accidents in the
  brazilian federal highways. These data are not provided in a clear and atractive way, and for that, a software is being developed
  as a proposal of a better presentation and use of these informations. Therefore, this work had as objective build an accident
  ontology for use in the software. The ontology construction followed the methodology 101. As an ongoing work, we obtained as partial 
  result the initial conceptual modeling of the ontology which will be used in the software.
  
\end{abstract}
     
\begin{resumo}
  
  A Polícia Rodoviária Federal (PRF) tem disponibilizado dados acerca dos acidentes ocorridos em rodovias federais.
  Esses dados não estão disponibilizados de forma clara e atrativa, para isso um \textit{software} está sendo desenvolvido como
  proposta de uma melhor apresentação e utilização das informações. Assim, este trabalho teve como objetivo construir uma ontologia de 
  acidentes para utilização no \textit{software}. A construção da ontologia seguiu a metodologia 101. Como trabalho em andamento, 
  obteve-se como resultado parcial a modelagem conceitual inicial da ontologia que posteriormente deve ser utilizada no \textit{software}.
  
\end{resumo}

\section{Introdução}
  
  Com a Lei de acesso à informação (LEI 12.527/2011) \cite{brasil11}, que garante
aos cidadãos o acesso a informações acerca dos órgãos públicos integrantes da
administração direta, a Polícia Rodoviária Federal (PRF) passou a disponibilizar dados
relacionados às ocorrências dos acidentes em rodovias federais. Todavia, são apenas
dados disponibilizados em planilhas. Desse modo, não sendo possível fazer interpretações
desses dados de forma fácil, eficiente e eficaz.

      Os dados sobre os acidentes fornecidos pela PRF, utilizados pelo sistema BR-Brasil, também servem de insumo para um \textit{software}, ainda 
      em construção, chamado ``Pé na estrada", que possui a finalidade de apresentar essas informações de forma mais atrativa, visual e significativa ao usuário.
      Atualmente, a aplicação é limitada às seguintes funcionalidades: buscar informações sobre as rodovias federais brasileiras,
      visualizando as rodovias com maiores índices de acidentes; traçar rotas para visualizar os trechos mais perigosos
      e os acidentes ocorridos ao longo da rota; e permite aos usuários fazerem comentários sobre as rodovias.
      
      Essa limitação é dada a partir do modelo de dados utilizado atualmente pela PRF, que consiste em uma estrutura relacional bastante complexa 
      e inconsistente, pois os relacionamentos entre os dados são feitos por meio de uma grande quantidade de tabelas relacionadas
      que limitam a representatividade semântica dos dados. Dessa forma, este trabalho visa construir uma ontologia de acidentes de trânsito baseada em ontologias já existentes para
possibilitar  uma melhor representação dos dados disponibilizados pela PRF através da aplicação ``Pé na estrada".

\section{Metodologia}
\label{metodologia_secao}

  A metodologia desse trabalho consistiu na adoção de um método para construção de ontologia
  e de uma revisão de literatura para o reúso de ontologias existentes. O método adotado para a construção dessa ontologia 
  foi o método \textit{\textbf{Ontology 101}} \cite{noy15}.

  De acordo com \cite{breitman05}, o processo de construção de uma ontologia a partir do método 101, resumidamente, 
  envolve as seguintes etapas:
    \begin{itemize}
    \item Definição das classes dessa ontologia;
    \item Disposição das classes em uma hierarquia taxonômica;
    \item Definição de propriedades e valores para os mesmos;
    \item Preenchimento dos valores das propriedades para cada instância.
    \end{itemize}

  Essas quatro etapas resumem os sete passos que podem ser vistos na Figura \ref{fig:processo_101}.
  \begin{figure}[ht] 
  \centering 
  \includegraphics[width=.9\textwidth]{11.png} 
  \caption[Processo de desenvolvimento de ontologias.]{Processo de desenvolvimento de ontologias. Fonte: \cite{DanielaLucas2008}}
  \label{fig:processo_101}
  \end{figure}

\section{Proposta de modelagem conceitual da ontologia}
  \label{ontologia_secao}
  
  Esta seção apresenta os resultados parciais obtidos com o processo de construção da ontologia,
  seguindo a metodologia apresentada na seção \ref{metodologia_secao}.
  
  \subsection{Escopo da ontologia}
    
      A ontologia proposta neste trabalho será utilizada para o domínio de acidentes envolvendo veículos ocorridos apenas em rodovias
      federais brasileiras, portanto, alguns conceitos inerentes a outros tipos de acidentes não se aplicam a este 
      contexto. 
%       Por esse motivo, os modelos conceituais das ontologias encontradas foram analisados e utilizados apenas como base 
%       para a construção da ontologia proposta. As ontologias encontradas com a revisão da literatura são apresentadas na
%       subseção \ref{ontologias_encontradas_secao}.
      
      Para a composição da ontologia, foram selecionados apenas algumas classes e atributos centrais para poder
      criar uma versão inicial da ontologia, com base nas ontologias encontradas e no modelo de dados atual
      do sistema BR-Brasil da PRF, pois tal modelo de dados conta com muitas classes e atributos.
    
  \subsection{Ontologias encontradas}
    \label{ontologias_encontradas_secao}
    
    Como mencionado, foi feita uma revisão da literatura a procura de ontologias relacionadas a acidentes de carro 
    por meio de pesquisas em bases de dados com expressões chaves, em inglês para obtenção de um resultado mais abrangente.
    
  \begin{center}
    Expressão 1: \textit{Car accident ontology}\\
    Expressão 2: \textit{Traffic accident ontology}
  \end{center}
  
    Foram encontradas duas ontologias bastante semelhantes, no contexto de acidentes de carro, que são empregadas em soluções tecnológicas parecidas, a 
    \textit{Car Accident Ontology for VANETs} (CAOVA) \cite{caova} e a \textit{Vehicular Accident Ontology} (VEACON) \cite{veacon}.
    
    Resumidamente, a aplicação dessas ontologias consiste na organização das informações de acidentes em estradas,
    para que as informações de um acidente de trânsito sejam devidamente transmitidas em redes veiculares 
    \textit{Ad hoc}, ou VANETs \cite{caova}.

  \subsection{Levantamento dos termos, classes e propriedades}
    
    Para levantar os termos inerentes ao contexto foi utilizada a técnica de \textit{CardSorting}.
    Essa técnica consiste na escrita de termos em cartões \cite{hudson}. Esses termos para a ontologia representam: a interação do usuário com a 
    aplicação,    quais seriam as perguntas que ele faria ao sistema e o que ele procuraria.

\section{Considerações finais}
  \label{consideracoes_secao}
  
  O presente trabalho apresentou os resultados do progresso parcial do processo de criação de uma ontologia para acidentes 
  em rodovias federais. Seguindo a metodologia adotada, a próxima etapa seria a criação de instâncias das classes definidas na
  ontologia, onde tais instâncias seriam feitas com base nos dados sobre os acidentes fornecidos pela PRF, para a posterior validação 
  da ontologia.
  
  Para trabalhos futuros, pretende-se pesquisar e utilizar ontologias específicas para as entidades como Pessoa (Ocupante),
  Veículo, Rodovia e Hospital para aumentar a representatividade da ontologia. Além disto, pretende-se incluir atributos mais específicos
  para cada classe, de modo a conseguir abrangir, com a ontologia, todos os dados necessários à PRF, fornecendo-lhes um modelo de dados 
  estruturado semanticamente.
     

      
      Com este trabalho, espera-se que a ontologia criada seja capaz de oferecer um arcabouço mais semântico para o \textit{software} 
      ``Pé na estrada", fornecendo um modelo de dados estruturado semanticamente que permita que as funcionalidades
      da aplicação evoluam, de modo a melhorar a apresentação e qualidade dos dados sobre os acidentes nas rodovias.
